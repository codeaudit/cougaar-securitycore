·         Updating user roles in the ontologies

To make a user role available to policy management, it must appear in the policy ontologies.  There are two relevant ontology files for this operation:  Ontology-ActorClasses.daml and Ontology-GroupInstances.daml.  These two files need to be updated in parallel.  For example, if one is adding a CAAdministrator role, then one must add the following to Ongology-GroupInstances.daml:

<ultralogGroup:Role rdf:ID="CAAdministratorRole"/>

and the following to Ontology-ActorClasses.daml:

<daml:Class rdf:ID="CAAdministrator">

    <rdfs:subClassOf rdf:resource="http://ontology.coginst.uwf.edu/Actor.daml#Person"/>

    <rdfs:subClassOf>

        <daml:Restriction>

            <daml:onProperty rdf:resource="http://ontology.coginst.uwf.edu/Group.daml#isMemberOf"/>

            <daml:hasValue rdf:resource="http://ontology.coginst.uwf.edu/Ultralog/Names/GroupInstances.daml#CAAdministratorRole"/>

        </daml:Restriction>

    </rdfs:subClassOf>

</daml:Class>
