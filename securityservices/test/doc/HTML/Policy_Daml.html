Servlet Access Control

·          DamlUriMap and Ontology-EntityInstances.daml

These two configuration files are used to control the relationship between the names of servlets that are understood by the policy and  the names of servlets that are used by the policy engine.  A new name for a servlet is introduced into the policy engine by adding a line of the following form to the file Ontology-EntityInstances.daml:

<ultralogEntity:Servlet rdf:ID="CAReadServlet"/>

The DamlUriMap file maps Ultralog names for servlets to the names used by the policy management system.  Each line in this file is either a comment (if it starts with #) or a pair of strings separated by a space.  A typical entry in this file looks like

/$*/CA/Index CAReadServlet

·          Updating user roles in the ontologies

To make a user role available to policy management, it must appear in the policy ontologies.  There are two relevant ontology files for this operation:  Ontology-ActorClasses.daml and.  These two files need to be updated in parallel.  For example, if one is adding a CAAdministrator role, then one must add the following to Ongology-GroupInstances.daml:

<ultralogGroup:Role rdf:ID="CAAdministratorRole"/>

and the following to Ontology-ActorClasses.daml:

<daml:Class rdf:ID="CAAdministrator">

    <rdfs:subClassOf rdf:resource="http://ontology.coginst.uwf.edu/Actor.daml#Person"/>

    <rdfs:subClassOf>

        <daml:Restriction>

            <daml:onProperty rdf:resource="http://ontology.coginst.uwf.edu/Group.daml#isMemberOf"/>

            <daml:hasValue rdf:resource="http://ontology.coginst.uwf.edu/Ultralog/Names/GroupInstances.daml#CAAdministratorRole"/>

        </daml:Restriction>

    </rdfs:subClassOf>

</daml:Class>
